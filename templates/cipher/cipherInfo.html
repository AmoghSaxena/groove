What is Symmetric primitives?

Overview

Symmetric cryptographic primitives, also known as secret-key cryptography, are fundamental building blocks in modern cryptosystems. They use a single secret key for both encryption and decryption, providing confidentiality, integrity, and authentication. Symmetric primitives are widely used due to their efficiency and performance compared to asymmetric primitives.

Key Features

  • Single secret key: Symmetric primitives use a single secret key that must be securely shared between the communicating parties.
  • Encryption and decryption: The same key is used for both encrypting plaintext into ciphertext and decrypting ciphertext back into plaintext.
  • Fast and efficient: Symmetric primitives are generally faster and more efficient than asymmetric primitives, making them suitable for encrypting large amounts of data.
  • Confidentiality: Symmetric encryption ensures that only parties with access to the secret key can decrypt and read the encrypted data.
  • Message authentication: Symmetric primitives can be used to generate message authentication codes (MACs) to ensure the integrity and authenticity of data.

Types of Symmetric Primitives

Symmetric primitives can be categorized into two main types: block ciphers and stream ciphers.

1. Block Ciphers

Block ciphers operate on fixed-size blocks of data, typically 64 or 128 bits. They encrypt each block independently using the secret key. Examples of block ciphers include:

  • AES (Advanced Encryption Standard): A widely used block cipher with key sizes of 128, 192, or 256 bits.
  • DES (Data Encryption Standard): A legacy block cipher with a 56-bit key size, now considered insecure.
  • Blowfish: A fast and secure block cipher with variable key sizes up to 448 bits.

2. Stream Ciphers

Stream ciphers encrypt data one bit or byte at a time, generating a pseudorandom keystream that is combined with the plaintext. Examples of stream ciphers include:

  • RC4 (Rivest Cipher 4): A widely used stream cipher, now considered insecure due to various vulnerabilities.
  • Salsa20: A fast and secure stream cipher with a 256-bit key and a 64-bit nonce.
  • ChaCha20: An improved variant of Salsa20, offering better security and performance.

3. Substitution Ciphers

These are classical ciphers are older encryption techniques that were used historically. While they are not secure by modern standards, they serve as important educational examples. Two notable classical ciphers are:

  • Caesar Cipher: A simple substitution cipher that shifts each letter in the plaintext by a fixed number of positions. It is a type of monoalphabetic substitution cipher.

4. Pseudorandom sequences

PRNGs are algorithms that generate sequences of numbers that appear random but are actually deterministic. They are used in various cryptographic applications, including key generation and nonce creation. One common PRNG is:
