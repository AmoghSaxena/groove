Simple Modular Hash Function

Simple Modular Hash Function

Overview

This is a good example of a keyless cryptographic primitive that can be demonstrated step-by-step is a simple hash function. Hash functions take an input message of any size and produce a fixed-size output called a hash or digest.

Real cryptographic hash functions like SHA-256 use more complex math to make it extremely difficult to find collisions or reverse the hash to recover the original input message. But they follow the same general principles of compressing an arbitrary input to a fixed-size output using math operations.

So in summary, hash functions are a fundamental building block in cryptography used for data integrity, commitment schemes, password verification and more. They are keyless primitives that map inputs to fixed-size outputs using mathematical formulas.

However, it's important to note that this hash function is still a simplified example for educational purposes and should not be used for real-world cryptographic applications. It is designed to be understandable for students who are new to hash functions and cryptography.

Properties

Even this simple hash has some important properties of cryptographic hash functions:

  • It's deterministic - the same input always produces the same output.
  • It compresses an arbitrary-sized input to a fixed-size output.
  • It's quick to compute the hash for any given message.

Formula

The hash function can be expressed as:

H(M) = SM(M)

Where:

  • H(M) is the hash value of message M
  • SM is the Simple modular hash function
  • M is the input message

Hashing Steps

  1. Convert the message to an integer using a rolling hash:

    • The formula x = (x * a + charCode) % m is used, where a is a multiplier (set to 31) and m is the modulus (set to 10000).
    • For each character in the message, the character code is obtained using charCodeAt(), and the rolling hash is updated using the formula.
    • The step-by-step calculation is displayed in the result.

  2. Apply additional mixing to the hash value:

    • The formula y = (y * y + i) % m is used, where y is initialized to the value of x from step 1, and i is the iteration number (0 to 4).
    • This mixing step is repeated 5 times to further scramble the hash value.
    • The step-by-step calculation is displayed in the result.

  3. Format the hash as a fixed-length string:

    • The final hash value y is converted to a string using toString().
    • The padStart() method is used to ensure a fixed length of 4 characters by padding with leading zeros if necessary.

  4. Convert the hash to a 16-digit binary representation:

    • The padded hash string is converted back to an integer using parseInt().
    • The toString(2) method is used to convert the integer to a binary string.
    • The padStart() method is used again to ensure a fixed length of 16 digits by padding with leading zeros.
